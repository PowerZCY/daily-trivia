What To Reuse

  • Visual tokens: gradients, blur, elevation, badge system, and button hierarchy
  • State model: current question, answer history, correct count, completion state
  • Analytics contract: start, answer, complete, archive click, report share
  • Persistence strategy: store completion per day or per content id

What To Replace Per Site

  • Brand palette and typography tokens
  • Question source and routing scheme
  • Analytics adapter implementation
  • Share copy and report text tone
Persistence

LocalStorage Usage

This sample also demonstrates how to persist quiz completion and archive-card state with browser storage.

Keys In This Sample

  • trivia-patterns-sample-completion: stores whether the daily sample has been completed, plus score and answer history
  • trivia-patterns-archive-completed: stores the list of archive days marked as completed

When It Reads And Writes

  • On page load: tries to restore a completed report from localStorage
  • On quiz completion: writes completion state and answer history
  • On archive completion: writes the finished day id into the completed-day list
  • On restart: removes the completion record and starts the sample fresh

Reference Flow

const STORAGE_KEYS = {
  sampleCompletion: "trivia-patterns-sample-completion",
  archiveCompleted: "trivia-patterns-archive-completed"
};

const saved = JSON.parse(localStorage.getItem(STORAGE_KEYS.sampleCompletion) || "null");
localStorage.setItem(STORAGE_KEYS.sampleCompletion, JSON.stringify({
  completed: true,
  correctCount,
  answerHistory
}));
